# deob Benchmark

Quantify deob's impact on LLM-based JavaScript reverse engineering through controlled experiments.

## Quick Start

```bash
# 1. To start from an existing experiment, copy it first:
cp -r experiments/<source> experiments/$(date -I)

# 2. Prepare scenarios: write original.js and ground-truth.json
#    - Write original.js with clean business logic
#    - Write ground-truth.json with expected answers
#    Place files in experiments/<date>/scenarios/<X>/

# 3. Generate obfuscated code
node experiments/<date>/tools/obfuscate.js

# 4. Run deob on all scenarios
node experiments/<date>/tools/runner.js

# 5. Spawn sub-agents for comparison (see Agent Prompt Template)
#    - Agent A reads deob output (0-prompt.md → 2-index.txt → main.js)
#    - Agent B reads obfuscated.js directly
#    - Save answers to results/scenario_<X>_deob.json and **_raw.json

# 6. Score results
node experiments/<date>/tools/score.js --all

# 7. Write report at experiments/<date>/report.md
```

## File Structure

```
benchmark/
  README.md                           ← this file
  experiments/                         ← all experiment runs
    2026-07-17-v1/                    ← one experiment (date-based naming)
      tools/                           ← experiment scripts
        obfuscate.js                   ← generate obfuscated.js from original.js
        runner.js                      ← run deob on all scenarios
        score.js                       ← compare agent answers to ground truth
      scenarios/                       ← experiment scenarios
        A/                             ← scenario A
          original.js                  ← original source code
          obfuscated.js                ← obfuscated version (generated by obfuscate.js)
          ground-truth.json            ← expected answers for scoring
        B/
        C/
        ...
      results/                         ← experiment output
        deob-output/                   ← deob output directories
          scenario_A/                  ← deob output for scenario A
            main.js                    ← deobfuscated code
            0-prompt.md                ← architecture overview for LLM
            1-structure.md             ← call graph and hotspots
            2-index.txt                ← function catalog
          scenario_B/
          ...
        scenario_A_deob.json          ← Agent A (deob) answer
        scenario_A_raw.json           ← Agent B (raw) answer
        scenario_B_deob.json
        scenario_B_raw.json
        ...
      report.md                       ← experiment report
```

## Scenario Requirements

Each scenario needs three files:

### 1. original.js

Clean, readable JavaScript code with meaningful business logic. Should include:
- Functions with clear purposes
- Call relationships between functions
- At least one "interesting" aspect (API call, security issue, complex logic)

### 2. ground-truth.json

Structured answers for scoring. Must match this format:

```json
{
  "scenario": "A",
  "difficulty": "easy|medium|hard",
  "description": "one sentence summarizing the code's purpose",
  "obfuscationTechniques": ["list of techniques used"],
  "functions": [
    {
      "name": "functionName",
      "purpose": "what it does (1 sentence)",
      "params": ["param1", "param2"],
      "returns": "return type description",
      "calls": ["fn1", "fn2"],
      "calledBy": ["fn3"],
      "security": ["security concern if any"]
    }
  ],
  "callGraph": {
    "functionName": ["list of functions it calls"]
  },
  "dataFlow": [
    "step 1 → step 2 → step 3"
  ],
  "securityIssues": [
    {"severity": "high|medium|low", "issue": "description", "location": "function name"}
  ],
  "apiEndpoints": [
    {"method": "GET|POST|PUT|DELETE", "path": "/api/endpoint", "purpose": "description"}
  ],
  "keyVariables": {
    "VARIABLE_NAME": "value or purpose"
  }
}
```

### 3. obfuscated.js

Generated by `tools/obfuscate.js`. Edit `obfuscate.js` to configure obfuscation options per scenario.

## Obfuscation Configuration

Edit `tools/obfuscate.js` to configure `javascript-obfuscator` options per scenario:

```javascript
const scenarios = {
  A: {
    name: "Scenario Name",
    difficulty: "easy",
    options: {
      renameGlobals: true,
      stringArray: true,
      stringArrayEncoding: ["base64"],  // or "rc4"
      controlFlowFlattening: true,
      controlFlowFlatteningThreshold: 0.5,  // 0.0-1.0
      deadCodeInjection: true,
      deadCodeInjectionThreshold: 0.2,
      selfDefending: true,
      debugProtection: true,
      splitStrings: true,
      numbersToExpressions: true,
      transformObjectKeys: true,
      unicodeEscapeSequence: true,
    }
  },
};
```

Key options and their impact:
| Option | Effect | Best for |
|--------|--------|----------|
| stringArray + stringArrayEncoding | Hides string literals | All scenarios |
| controlFlowFlattening | Reorders branches into switch-case | Medium/Hard |
| deadCodeInjection | Adds unreachable code blocks | Medium/Hard |
| selfDefending | Anti-tampering checks | Hard |
| debugProtection | DevTools detection | Hard |

## Agent Prompt Template

Use this template when spawning sub-agents:

**Agent A (deob)**:
```
RE task: Analyze deobfuscated JavaScript code.

READ IN ORDER:
1. {deob_output_dir}/0-prompt.md
2. {deob_output_dir}/2-index.txt (full file)
3. {deob_output_dir}/main.js (focus on banners)

Answer:
{
  "purpose": "one sentence",
  "functions": [{"name":"...","purpose":"...","calls":[...],"calledBy":[...]}],
  "endpoints": [{"method":"...","path":"...","purpose":"..."}],
  "security": [{"severity":"high|medium|low","issue":"..."}],
  "dataFlow": "input → processing → output",
  "variables": [{"name":"...","value":"..."}],
  "entryPoint": "...",
  "functionCount": 0
}
JSON only, no markdown.
```

**Agent B (raw)**:
```
RE task: Analyze OBFUSCATED JavaScript code directly.

READ: {scenario_dir}/obfuscated.js

Answer: (same JSON format as Agent A)
JSON only, no markdown.
```

Save agent answers as:
- `results/scenario_{X}_deob.json` (Agent A)
- `results/scenario_{X}_raw.json` (Agent B)

## Scoring

```bash
# Score specific scenarios
node tools/score.js A B

# Score all scenarios
node tools/score.js --all

# Override experiment dir
BENCH_EXP_DIR=/path/to/experiment node tools/score.js --all
```

### Scoring Weights
| Dimension | Weight | Method |
|-----------|--------|--------|
| Functions | 30% | F1 score (name + purpose matching) |
| Security | 20% | Issue keyword overlap |
| Endpoints | 15% | Exact method + path matching |
| Purpose | 10% | Keyword overlap with description |
| DataFlow | 10% | Keyword overlap |
| Variables | 10% | Value/purpose matching |
| EntryPoint | 5% | Existence check |
